Concepts

  1. Tag markup: markup which function is, basically, to create elements and orders to organize, identify and limit content in a website. Then, the browser comes, and automatically, recognizes and renders the .html file, so that the user can read the info displayed;
  2. Element: everything from the start tag to the end tag;
  3. Attribute tag: informations that provide additional or special characteristics/attributes about elements. Can also act as a kind of shortcut to modify an element, without having to do a lot of work.;
  4. Anchor tag: tag used only to link different webpages, attach documents or even create shortcuts in the current webpage;
  5. URL: Uniform Resource Locator (or just URL) is the adress to get to an webpage. All websites and webpages have an unique name or link ("google" or "www.google.com", for example);

Attribute Tags

Attribute structure: name of the attribute = its value, that can be a word or number, genereally inside quotes. Have to be only in the oppening tag.

< [openning tag] + [name="value"] >

    Common Global Attributes (can be used with all HTML elements)

  • id="any name, no space, no special characters": give identity only for a
    single tag, then it can be modified by css

  • class="any name, same rules as id": give identity for tags, a unique css can be applied for many tags
  • style="a css rule": style shortcuts, instead of oppening css selectors

  • lang="abbreviation of the idiom used-abbreviation of country": not necessary but helps in search, goes inside html tag, have to specify,
    1. language - e.g.: en | pt | es
    1.1 country (otional)- e.g.: US | BR | CA

  • title="extra information": displayed as a tooltip, when the user mouse over the element

  • align="position": text alignment, alternative for css

  • hidden: specifies that an element is not yet, or is no longer, relevant

    • Common Specific Attributes

  • href="URL destination": with anchor tag and others, must specify,
    1. URL of the page the link goes (absolute URL) - e.g.: href="https://google.com"
    2. or path of the other local .html file (relative URL) - e.g.: href="user/downloads/images/image1.html"
    3. or specific section of the current webpage (the id attribute) - e.g.: href="(id destination)"

  • src="any kind of media": have to specify,
    1. relative URL - e.g.: src="user/downloads/images/image1.jpg"
    2. or absolute URL - e.g.: src="https://iamge1.png"
    3. or only the file name (if the document and media are in the same folder) - src="image1.jpg"

  • width="measure" + height="measure": usually comes with src, plus have to specify
    1. measures with units

  • alt="alternative text": in case the image does't loads and other specific things, there will be an text
